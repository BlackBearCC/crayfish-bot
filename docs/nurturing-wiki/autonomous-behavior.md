# 自主行为系统

> 更新: 2026-03-09

[← 返回首页](index.md)

## 设计原则

宠物不应该只在用户操作时才有反应。它应该**自己找事做**——在你不理它的时候也像个活物。这是"陪伴感"的核心来源。

---

## 行为状态机

```
                    ┌──────────┐
                    │  idle    │ ← 默认状态
                    └────┬─────┘
                         │ 定时器 / 条件触发
          ┌──────────────┼──────────────┐
          ▼              ▼              ▼
    ┌──────────┐  ┌──────────┐  ┌──────────┐
    │ wander   │  │ perform  │  │ react    │
    │ 自由走动  │  │ 表演动作  │  │ 环境反应  │
    └──────────┘  └──────────┘  └──────────┘
          │              │              │
          └──────────────┴──────────────┘
                         │
                    回到 idle
```

- **idle**: 站着/坐着/小幅度待机动画
- **wander**: 自主移动，探索桌面
- **perform**: 自发做一个有趣的动作
- **react**: 对环境变化做出反应

## 自主行为列表

### 日常行为 (高频，每 1-5 分钟随机触发)

| 行为 | 动画 | 触发条件 |
|------|------|---------|
| 打哈欠 | 张嘴伸懒腰 | idle 超过 3 分钟 |
| 挠痒痒 | 后腿挠耳朵 | 随机 |
| 舔毛 | 低头舔爪子 | 随机 |
| 伸懒腰 | 前爪伸展 | idle 后起身时 |
| 甩尾巴 | 尾巴摇动 | 心情好时 |
| 东张西望 | 头左右转 | idle 时随机 |
| 坐下 | 从站立变坐姿 | idle 超过 2 分钟 |
| 趴下 | 从坐变趴 | idle 超过 5 分钟 |

### 自主探索 (中频，每 5-15 分钟)

| 行为 | 描述 | 条件 |
|------|------|------|
| 散步 | 在当前平台来回走 | 随机 |
| 跳窗口 | 跳到另一个窗口顶部 | 有可达窗口 |
| 下地 | 从窗口跳到桌面 | 随机 |
| 爬上去 | 从桌面跳上窗口 | 随机 |
| 边缘探头 | 走到屏幕边缘探出去 | 走到边界时 |
| 巡逻 | 沿窗口顶部走完整一遍 | 站在大窗口上时 |

### 情绪驱动行为 (根据属性状态)

| 心情 | 行为 | 动画 |
|------|------|------|
| 高 (>70) | 哼歌 | 摇头晃脑，头顶冒音符 |
| 高 | 跳舞 | 原地小幅扭动 |
| 高 | 翻滚 | 在平台上翻个跟头 |
| 中 (40-70) | 正常日常行为 | — |
| 低 (<40) | 叹气 | 低头，头顶冒省略号 |
| 低 | 蜷缩 | 缩成一团 |
| 低 | 发呆 | 坐着不动，眼神空洞 |

| 饱腹 | 行为 | 动画 |
|------|------|------|
| 饿 (<60) | 翻找 | 低头到处嗅/扒拉 |
| 饿 | 盯着用户 | 转向鼠标方向，眼巴巴看着 |
| 饱 (>225) | 打饱嗝 | 拍肚子 |
| 饱 | 犯困 | 打哈欠频率增加 |

## 时间感知

宠物根据系统时间改变行为模式。

| 时段 | 行为模式 |
|------|---------|
| 6:00-9:00 早晨 | 起床伸懒腰、打哈欠、活跃度逐渐上升 |
| 9:00-12:00 上午 | 正常活跃，探索/玩耍频率高 |
| 12:00-13:00 午间 | 犯困、趴下概率增加 |
| 13:00-18:00 下午 | 正常活跃 |
| 18:00-21:00 傍晚 | 活跃度下降，坐/趴增加 |
| 21:00-23:00 夜晚 | 打哈欠频繁，催你"该休息了" |
| 23:00-6:00 深夜 | 睡觉状态，偶尔翻身/说梦话气泡 |

## 用户活动感知

### 键盘/鼠标活跃度

```
检测逻辑 (每 30s 采样一次):
  - 过去 30s 内有键盘/鼠标事件 → 用户活跃
  - 过去 5min 无事件 → 用户离开

用户活跃时:
  - 宠物安静陪伴，减少大动作 (不打扰工作)
  - 偶尔抬头看看用户方向

用户离开时:
  - 宠物自由活动，探索/玩耍频率升高
  - 用户回来时 → 宠物跑过来迎接 (跑向鼠标位置)
```

### 应用感知 (可选)

检测当前前台应用，宠物做出不同反应：

| 应用类型 | 宠物反应 |
|---------|---------|
| IDE / 代码编辑器 | 安静坐在旁边，偶尔看看代码方向 |
| 浏览器 | 好奇探头 |
| 游戏 | 拿爆米花坐下看 |
| 视频播放 | 跟着看，偶尔回头 |
| 长时间不切换 | "你还在吗?" 气泡 |

实现：读取前台窗口标题/进程名，匹配关键词分类。隐私安全——只读窗口标题，不读内容。

## 行为调度器

```typescript
interface AutonomousBehavior {
  id: string;
  animation: string;           // 对应的动画名
  duration: number;            // 持续时间 ms
  cooldown: number;            // 最小间隔 ms
  weight: number;              // 权重 (受心情/饱腹/时间影响)
  condition?: () => boolean;   // 额外触发条件
  platform?: "any" | "window" | "ground" | "taskbar";
}

class BehaviorScheduler {
  private lastBehaviorAt: Record<string, number> = {};

  // 每 10-30s 调用一次 (在 idle 状态时)
  tick(): AutonomousBehavior | null {
    const candidates = ALL_BEHAVIORS
      .filter(b => this.isOffCooldown(b))
      .filter(b => !b.condition || b.condition())
      .filter(b => this.platformMatch(b));

    if (candidates.length === 0) return null;

    // 加权随机选择
    const weights = candidates.map(b => this.adjustWeight(b));
    return weightedRandom(candidates, weights);
  }

  // 根据宠物状态调整权重
  private adjustWeight(b: AutonomousBehavior): number {
    let w = b.weight;
    // 心情高 → playful 行为权重 x2
    // 心情低 → sad 行为权重 x2
    // 深夜 → sleep 行为权重 x5
    // 用户活跃 → 安静行为权重 x3
    return w;
  }
}
```

## 行为优先级

当多个触发条件同时满足时：

```
用户交互 > 环境反应 > 情绪驱动 > 自主探索 > 日常行为 > idle
```

- 用户点击/拖拽 → 立即中断当前行为
- 窗口关闭(脚下平台消失) → 中断，执行掉落
- 其他行为播放完毕才切换，不打断

## 和养成系统的连接

自主行为不是独立模块，它受养成状态驱动：

| 养成状态 | 对行为的影响 |
|---------|------------|
| 心情高 | 活泼行为增多（跳舞/翻滚/哼歌） |
| 心情低 | 消沉行为增多（叹气/蜷缩/发呆） |
| 饱腹低 | 觅食行为（翻找/盯着用户） |
| 等级高 | 解锁更多行为种类 |
| 亲密度高 | 主动靠近用户(鼠标)的频率增加 |
| 健康低 | 行动迟缓，活动频率降低 |

## 动画需求清单

基于以上行为，需要的动画列表：

**基础 (必须有):**
idle_stand, idle_sit, idle_lie, walk, run, jump, fall, land

**日常 (核心体验):**
yawn, scratch, lick, stretch, tail_wag, look_around

**情绪:**
happy_dance, happy_hum, happy_roll, sad_sigh, sad_curl, sad_stare

**桌面交互:**
climb_edge, peek_edge, hang_wall, slide_down, push_window, sniff_cursor, startled, wave

**拖拽反应:**
grabbed, dizzy, thrown, crawl_back

**状态:**
sleep, sleep_turn, eat, sick

**总计约 30 种动画**，每种 4-12 帧。这是商业桌宠的标准量级。
