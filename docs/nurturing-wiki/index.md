# 宠物养成系统 Wiki

> 版本: v2.1 | 日期: 2026-03-09

## 现状问题

| 系统 | 状态 | 问题 |
|------|------|------|
| 属性系统 (mood/hunger/health) | 有衰减，值域 0-100 | 只降不升，无主动恢复手段 |
| 成长系统 (intimacy) | 只增不减，驱动阶段进化 | 仅靠交互获取，无养成感 |
| 技能系统 (5维属性+7领域) | 通过对话/工具使用积累 XP | 与养成脱节，无法影响宠物状态 |
| 学习系统 (课程+碎片) | 在线计时，完成得 XP | 独立封闭，不影响核心循环 |
| 成就系统 (12枚徽章) | 条件检测 + 解锁 | 奖励仅是亲密度，无实质激励 |

**核心缺失:**
1. 数值只降不升 — 离线8小时后数值见底
2. 没有宠物等级 — intimacy 阶段更像关系，不是成长等级
3. 没有养成循环 — 用户无法通过"照顾"获得正反馈

## 核心循环

```
┌──────────────────────────────────────────────────┐
│                  核心循环                          │
│                                                    │
│   喂食(补充 token)  ─→  饱腹值上升                  │
│         ↑                    ↓                     │
│     获得食物道具         聊天消耗 token              │
│         ↑                    ↓                     │
│     每日任务/升级       饱腹值下降 + 亲密度/心情变化   │
│         ↑                    ↓                     │
│      EXP 增长  ←─── LLM 评估(每5条) ──→ mood/intimacy │
└──────────────────────────────────────────────────┘
```

## 实现优先级

### P0 — 核心循环 (第一期)

1. **等级系统** — EXP + 升级 + 等级显示
2. **聊天消耗 hunger** — 每条消息 -10 hunger，工具调用额外 -1，hunger 不足时拒绝聊天
3. **基础喂食(猫粮)** — 免费补充 token，冷却 20min
4. **数值再平衡** — hunger max 300, 衰减降至 0.3/min, 离线保护, 底线 60
5. **交互入口改造** — 右键精简 3 项 + 养成面板 4-tab

完成后效果: 聊天消耗饱腹，喂食补充饱腹，宠物有等级，养成面板一站式管理。

### P1 — LLM 评估 + 任务 + 商城 (第二期)

6. **聊天 LLM 意图提取** — 每5轮+间隔>=5min 双条件触发
7. **每日任务系统** — 后端档位 + LLM 生成描述 + 奖励发放(含星币)
8. **道具背包 + 商城** — 背包 UI + 商城 UI + 星币货币
9. **玩耍/休息/治疗** — 更多养护手段

### P2 — 长线内容 (第三期)

10. **连续登录** — 签到 + 累计奖励
11. **等级解锁内容** — 新动画/外观/自动照顾
12. **等级 x 亲密度联动** — persona 增强

## 体验节奏

```
08:00  开机 → 离线底线 hunger 60 → 气泡"有点饿了" → 右键喂一口(+75) → 刷每日任务
09:00  聊天 → 每条 hunger -10 + mood +1 → 第5条触发 LLM 评估 → 第8条 hunger 偏低
09:30  喂一口(CD好了) +75 → 完成 easy 任务 → EXP +8, 星币 +12
12:00  商城买巴别鱼罐头(30星币) → 完成 medium 任务 → EXP +15, 星币 +25
14:00  长按抚摸 → LLM 连续识别 praise → streak 加成越来越开心
18:00  完成 hard 任务 → EXP +25, 星币 +45 → 今日总 EXP ~100, 星币 ~120
23:00  关机 → 明天属性不归零，继续养成
```

## 兼容性

- **固定交互数值保留**: click/longpress/feed/file_drop 的固定 mood/intimacy 不变，LLM 评估是叠加层
- **LLM 角色明确**: 只做意图分类，不做数值决策; 程序查表+二次函数
- **不影响 skill/learning/achievement**: 保持独立，产出 EXP 和道具联动
- **安全兜底**: 未知意图 fallback neutral; LLM 失败静默跳过; clamp 保底
- **渐进式实现**: 每期独立可交付
