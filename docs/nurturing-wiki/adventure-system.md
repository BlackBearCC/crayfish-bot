# 探险系统

> 状态：v2 重新设计（线索卡 + 途中事件）
> 更新：2026-03-13

---

## 设计理念

灵感来源：暗黑地牢马车遭遇。去掉"填表配置"式的启动流程，改为**选卡出发 + 途中随机事件 + 结算叙事**，轻量但有惊喜感。

核心原则：
- **最少操作**：玩家只需选一张线索卡即可出发，零配置
- **途中有事发生**：探险不是纯等待，过程中会弹出事件
- **宠物有自主性**：玩家不操作时，宠物自己做决定
- **AI 驱动叙事**：所有文案由 LLM 生成，每次探险体验不同

---

## 核心流程

```
打开探险 Tab
    │
    ▼
展示 3 张线索卡（LLM 生成，卡牌视觉）
    │  选一张卡
    ▼
LLM 生成开场叙事 + 预生成途中事件，探险开始
    │
    ▼
途中触发 1-3 个随机事件（narration / choice / discovery）
    │
    ▼
时间到 → LLM 生成结局叙事 + 发放奖励
    │
    ▼
全平台通知（WebSocket + Telegram/Discord 等）
```

## 线索卡（Rumor Card）

### 生成机制

- 打开探险 Tab 时，调用 LLM 一次性生成 3 张线索卡
- 每张卡包含：钩子文案（1 句话）、地点、风险等级（⭐1-3）、预估时长、氛围标签
- LLM 根据宠物当前状态/性格/历史探险生成，保证多样性
- 可刷新换一批（消耗少量金币，防止无限刷）

### 数据结构

```typescript
interface RumorCard {
  id: string;
  hook: string;        // "森林深处传来奇怪的歌声..."
  location: string;    // "神秘森林"
  risk: 1 | 2 | 3;    // 星级，对应 safe/moderate/dangerous
  duration: number;    // 分钟
  theme: string;       // 氛围色标签: "forest" | "ruin" | "water" | "cave" | "town" | "sky"
}
```

### 风险与奖励

| 星级 | 风险 | 基础成功率 | EXP | 金币 |
|------|------|-----------|-----|------|
| ⭐ | safe | 90% | 20 | 10 |
| ⭐⭐ | moderate | 70% | 50 | 25 |
| ⭐⭐⭐ | dangerous | 50% | 100 | 50 |

## 途中事件（Encounter）

### 事件类型

原来的三种探险类型下沉为途中事件类型：

| 类型 | 说明 | 玩家操作 |
|------|------|---------|
| **narration**（叙事） | 纯故事推进，描述旅途见闻 | 无，自动展示 |
| **choice**（抉择） | 遇到岔路/困境，需要 A/B 选择 | 可选，超时宠物自决 |
| **discovery**（发现） | 发现道具/线索/NPC | 无，直接获得奖励 |

### 触发规则

- 根据探险时长和风险等级决定事件数量：
  - ⭐ (≤5min)：1 个事件
  - ⭐⭐ (5-10min)：1-2 个事件
  - ⭐⭐⭐ (>10min)：2-3 个事件
- 事件在探险时间轴上均匀分布（如 5 分钟探险，事件在第 2-3 分钟触发）
- 事件类型随机，但至少保证 1 个非 narration 类型（⭐⭐ 以上）

### 数据结构

```typescript
interface Encounter {
  id: string;
  type: "narration" | "choice" | "discovery";
  text: string;             // 事件描述
  choices?: {               // 仅 choice 类型
    a: string;
    b: string;
  };
  selectedChoice?: "a" | "b";  // 玩家选择，或宠物自决
  petDecided?: boolean;     // 是否宠物自己选的
  reward?: {                // 仅 discovery 类型
    item?: string;
    coins?: number;
  };
  triggeredAt?: number;     // 触发时间戳
  resolvedAt?: number;      // 解决时间戳
}
```

### 抉择超时

- 抉择事件弹出后，玩家有 **60 秒** 做选择
- 超时 → 宠物自己决定（随机或 LLM 选，标记 `petDecided: true`）
- 选择影响最终成功率（好选择 +10%，坏选择 -10%）

## 探险生命周期

### 状态流转

```
idle → exploring → completed
                 ↘ cancelled
```

### 数据结构（v2）

```typescript
interface Adventure {
  id: string;
  card: RumorCard;          // 选中的线索卡
  status: "exploring" | "completed" | "cancelled";
  encounters: Encounter[];  // 途中事件列表（LLM 预生成）
  nextEncounterIdx: number; // 下一个待触发的事件索引
  story?: string;           // LLM 开场叙事
  result?: AdventureResult; // 结算结果
  createdAt: number;
  startedAt: number;
  endedAt?: number;
}
```

### tick() 逻辑

每次心跳检查：
1. 是否有待触发的事件？→ 触发，广播给客户端
2. 抉择事件是否超时？→ 宠物自决
3. 总时长是否到了？→ 进入结算

## RPC 接口（v2）

| 方法 | 说明 |
|------|------|
| `character.adventure.rumors` | 获取 3 张线索卡（LLM 生成） |
| `character.adventure.start` | 选卡出发，返回探险 + 开场叙事 |
| `character.adventure.choice` | 对抉择事件做选择 |
| `character.adventure.active` | 获取当前探险状态（含已触发事件） |
| `character.adventure.cancel` | 取消探险 |
| `character.adventure.history` | 历史记录 |

移除的接口：
- ~~`character.adventure.complete`~~ — 不再需要手动完成，tick 自动结算

## 客户端 UI 设计

> 视觉基调：沿用项目手绘赛璐璐风格（`--radius-sketchy`、`--shadow-ink`、`--border-ink`），
> 但探险卡牌要有**独立的游戏感**——比养成面板其他 Tab 更强的视觉冲击。

### 设计语言

- **卡牌 = 核心交互单元**。线索卡不是列表项，是真正的"卡片"——有厚度感、可翻转暗示、hover 上浮
- **氛围色彩**：每张卡根据 `theme` 字段有不同的渐变底色（森林绿、废墟灰、水域蓝、洞穴紫、城镇暖黄、天空淡蓝），不是千篇一律的白底
- **星级 = 视觉张力**：⭐ 卡片平和舒缓，⭐⭐⭐ 卡片有更浓的色彩和更重的阴影，传递危险感
- **手绘边框**：继承 `--radius-sketchy`，但卡牌用更大的不规则圆角
- **微动效**：卡片 idle 时轻微呼吸（scale pulse），hover 上浮 + 阴影加深，点击压下反馈

### 主题色板（theme → 渐变）


#### 卡片交互细节

| 状态 | 效果 |
|------|------|
| idle | `scale(1)` + 轻微 breathing 动画（0.5s ease `scale(1) → scale(1.005) → scale(1)`），3张卡片错开节奏 |
| hover | `translateY(-3px)` + `box-shadow` 加深 + 边框色加深，0.2s transition |
| click/tap | `scale(0.97)` 压下 → 松手出发 → 卡片展开为进行中视图（0.3s） |
| loading | 卡片内容替换为骨架屏 shimmer（3 张空卡同时闪烁） |
| disabled | 正在生成线索卡时，3 张位置用骨架卡片占位 |

#### 星级视觉层级

| 星级 | 边框 | 阴影 | 额外效果 |
|------|------|------|---------|
| ⭐ | `2px solid var(--ink)` | `--shadow-ink-light` | 无 |
| ⭐⭐ | `2.5px solid var(--ink)` | `--shadow-ink-sm` | 无 |
| ⭐⭐⭐ | `3px solid var(--ink)` | `--shadow-ink` + 底部红色微光 | 卡片底部 `border-bottom: 3px solid #ef4444` |

#### 事件节点样式

| 事件状态 | 节点 | 卡片 |
|----------|------|------|
| 已触发 — narration | 灰色实心圆 | 浅灰底，斜体文字 |
| 已触发 — discovery | 金色实心圆 | 金色边框，道具名 + icon |
| 当前 — choice（等待选择） | 橙色脉冲圆 | 白底重边框，A/B 按钮突出 |
| 已触发 — choice（已选） | 绿色实心圆 | 高亮所选项，另一项灰显 |
| 未触发 | 空心虚线圆 | "???" 占位，虚线边框 |

#### 抉择按钮（A/B）

- 两个并排按钮，各占 48% 宽度
- 手绘边框，hover 底色变化
- 按钮上方小字倒计时 "⏱️ 48s"
- 超时动画：倒计时归零 → 按钮同时变灰 → 显示"宠物选了 A"

### 气泡通知

途中事件同时以气泡形式弹出在桌面宠物上方（不仅在面板内）：
- narration → 普通文字气泡（`StreamingBubble` 现有机制），3s
- choice → **特殊气泡**：带 A/B 两个小按钮 + 倒计时文字，60s
- discovery → 带金色边框的气泡 + 道具名，3s

### 骨架屏 / 加载态

- 线索卡加载中：3 张空白卡片，内部用 shimmer 动画（浅灰 → 白 → 浅灰 横向扫过）
- 探险开场生成中：卡片展开后内容区域 shimmer + "生成探险故事中..." 文字
- 事件触发时：节点脉冲 0.3s 后内容淡入

## 与养成系统联动

- 探险消耗 hunger（出发时 -20）
- 探险失败扣 health（damage: 10）
- 探险成功加 mood（+15）
- 探险获得 EXP 计入成长系统
- discovery 事件获得的道具进入背包
- 探险历史影响线索卡生成（避免重复地点）
