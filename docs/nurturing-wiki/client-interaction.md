# 客户端交互系统

> 更新: 2026-03-09

[← 返回首页](index.md)

宠物不是"浮在桌面的窗口"，而是**住在你桌面上的生物**。

本文分两部分：**桌面感知**（窗口/鼠标/物理）和 **UI 面板**（菜单/养成面板/气泡）。

---

# 一、桌面感知

## 窗口感知

### 窗口边缘站立

宠物能"看见"其他窗口的边界，站在窗口顶部边缘。

```
┌─────────────────────┐
│  VS Code             │
│                      │  😺 ← 站在窗口顶上
├──────────────────────┤
│  code code code      │
```

- Win32 API (`EnumWindows` / `GetWindowRect`) 获取可见窗口列表
- 宠物可站/走/坐在任意窗口的顶部边缘
- 窗口移动/关闭时宠物跟随或跳下
- 窗口最大化时宠物跳到任务栏或桌面底部

### 任务栏栖息

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  🪟  📁  🌐  💬       😺        🔊 🕐
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- 检测任务栏位置和高度（支持上/下/左/右）
- 在任务栏区域上方行走/坐下
- 用户点击任务栏图标时宠物可能被"吓一跳"

### 屏幕边缘交互

| 行为 | 触发 | 动画 |
|------|------|------|
| 探头 | 宠物走到屏幕边缘 | 身体藏在边缘后面，只露出头 |
| 爬上来 | 从屏幕下方/侧边 | 双手扒住边缘，慢慢爬上来 |
| 挂壁 | 走到屏幕侧边 | 像壁虎一样贴在边上 |
| 滑落 | 挂壁一段时间后 | 慢慢往下滑，抓不住 |

## 鼠标交互

### 追鼠标

当鼠标静止超过 3 秒且在宠物附近（200px 内），宠物好奇地走向鼠标。

```
状态机:
  idle → 检测鼠标距离 < 200px 且静止 > 3s
       → curious_walk (朝鼠标方向走)
       → 到达鼠标位置 → sniff (闻一闻) → idle
       → 鼠标移动 → startled (吓一跳) → idle
```

- CD 30s，不能太频繁
- 用户忙碌打字时不触发（检测键盘活跃度）

### 躲鼠标

低心情时，鼠标靠近宠物，宠物会主动躲开。

### 玩鼠标

高心情时，宠物会试图"拍"鼠标指针（像猫拍激光笔）。

## 窗口互动

### 推窗口

```
触发条件:
  - 宠物处于 playful 状态
  - 随机触发 (概率 5%，CD 10min)
  - 目标窗口非全屏、非置顶

效果:
  - 窗口水平移动 3-8px
  - 宠物播放 push 动画
  - 推完坐下休息
```

轻微到不影响工作，但能被注意到。可在设置中关闭。

### 窥屏

宠物走到窗口边缘，转身看向窗口内容，做出好奇/无聊的表情。

### 拍窗口

宠物站在窗口顶部，用爪子拍窗口标题栏。纯动画，无实际效果。

## 物理系统 (简化版)

### 重力

宠物默认受重力影响，会落到最近的"平台"上：

```
平台优先级:
  1. 当前站立的窗口顶部
  2. 下方最近的窗口顶部
  3. 任务栏顶部
  4. 屏幕底部 (地面)
```

### 跳跃

```
跳跃条件:
  - 目标窗口顶部与当前平台高度差 < 200px
  - 水平距离 < 300px

动画: 蓄力 → 抛物线跳跃 → 着陆
```

### 掉落

窗口关闭/移开时，宠物脚下平台消失 → 掉落动画 → 落到下一个平台。

### 拖拽物理

| 拖拽行为 | 反应 |
|---------|------|
| 抓起 | 四肢张开，表情惊讶 |
| 快速移动 | 表情变晕 |
| 甩动 | 眼睛变螺旋 |
| 放下 | 轻放 → 站稳；高处扔下 → 掉落+着陆 |
| 扔向屏幕边缘 | 从边缘滑出去，过几秒从另一边爬回来 |

## 技术方案

### 平台感知层

```typescript
interface DesktopPlatform {
  id: string;          // 窗口句柄 or "taskbar" or "screen_bottom"
  rect: { x, y, w, h };
  type: "window" | "taskbar" | "ground";
}

// 定期刷新 (每 1-2s)
function scanPlatforms(): DesktopPlatform[] {
  // Win32: EnumWindows + GetWindowRect
  // macOS: CGWindowListCopyWindowInfo
  // 过滤: 排除自身窗口、不可见窗口、全屏窗口
}
```

### Electron 实现要点

- 透明无框窗口 (现有)
- 窗口大小需要覆盖宠物可活动范围
  - 方案 A: 全屏透明窗口，宠物在内部移动 (简单但覆盖点击)
  - 方案 B: 窗口跟随宠物移动 + 扩大碰撞区域 (现有方案改进)
- 点击穿透: 非宠物区域设置 `pointer-events: none`
- 鼠标位置: `screen.getCursorScreenPoint()` 轮询 (100ms)
- 窗口列表: 通过 Node addon 或 child_process 调用系统 API

### 性能约束

- 平台扫描: 每 2s 一次
- 鼠标追踪: 100ms 轮询，仅在需要时开启
- 宠物物理: 和渲染同帧 (`requestAnimationFrame`)
- 总 CPU 占用目标: < 2%

# 二、UI 面板

## 设计原则

右键菜单只放**最高频动作**，复杂功能收进**养成面板**。不在菜单里堆功能。

---

## 右键菜单 (3 项)

```
┌───────────────┐
│  🍖 喂一口     │  ← 42号口粮一键喂食 (含冷却检查)
│  📋 养成面板    │  ← 打开完整面板
│  ⚙️ 设置       │
└───────────────┘
```

- "喂一口"最高频操作，一键触达
- 冷却中显示剩余时间灰态，不隐藏
- 商城/背包/任务/玩耍/治疗 → 全部通过养成面板

## 状态气泡规则

宠物头顶气泡 = **被动提示**，不需用户主动查看。

| 条件 | 气泡内容 | 优先级 |
|------|---------|--------|
| hunger <= 60 | "有点饿了~" | 低 |
| hunger <= 30 | "太饿了...先喂喂我吧" (拒绝聊天) | 高 |
| health < 40 | "不太舒服..." | 中 |
| mood < 30 | "心情不太好..." | 中 |
| 升级时 | "升级啦! Lv.N!" | 即时 |
| 任务完成 | "任务完成! 快来领奖~" | 即时 |

自动消失 (3-5s)，同时只显示最高优先级的一条。
